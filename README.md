# dev-2026-01

## 题目名称

HTML + CSS 布局与样式综合开发：实现一个新人可完成的任务管理工作台页面

## 题目背景

请你使用 **HTML + CSS** 完成一个“任务管理工作台”静态页面。

这是一道面向新人的前端基础考核题，重点考察：

- HTML 语义化结构
- CSS 基础布局能力
- Flex / Grid 的实际使用
- 常见组件样式实现能力
- 响应式适配能力
- 基本代码组织能力

本题不要求接入后端，不要求实现复杂 JavaScript 逻辑。

---

## 开发要求

请完成一个静态页面，页面至少包含以下部分：

### 1. 顶部区域
需要包含：

- 页面标题或产品名称
- 一个搜索框
- 一个“新建任务”按钮
- 一个用户信息区域

### 2. 侧边导航
至少包含 4 个导航项，例如：

- Dashboard
- My Tasks
- Projects
- Settings

要求：

- 当前选中项有明显样式
- 鼠标悬停时有样式变化

### 3. 主内容区
至少包含以下两个模块：

#### A. 数据概览卡片
至少 3 张卡片，例如：

- 今日待办
- 进行中任务
- 已完成任务

#### B. 任务列表
至少展示 5 条任务，且每条任务至少包含：

- 任务名称
- 所属项目
- 截止日期
- 状态

状态至少区分 2 种，例如：

- 进行中
- 已完成
- 待开始

### 4. 附加信息区域
从下面任选 1 种实现即可：

- 今日日程
- 最近活动
- 团队成员
- 即将到来的任务

---

## 布局要求

### 桌面端
- 页面结构清晰
- 侧边栏、主内容区层次明确
- 卡片和任务列表排版整齐

### 移动端
当屏幕宽度较小时，页面需要：

- 保持主要内容可正常阅读
- 卡片和列表能够合理换行或改为纵向排列
- 不出现明显重叠、溢出、错位

---

## 技术要求

### 必须满足

- 使用原生 HTML + CSS 完成
- 至少使用 1 次 `flex`
- 至少使用 1 次 `grid`
- 使用合理语义标签，例如：
  - `header`
  - `aside`
  - `main`
  - `section`
  - `nav`
- 有基础交互状态样式：
  - `:hover`
  - `:focus`

### 不建议做法

以下做法会影响评分：

- 大量使用 `position: absolute` 硬拼布局
- 整个页面几乎全靠 `div`
- 直接套用完整后台模板
- 不做移动端适配

- 移动端表现正常，10 分

- CSS 结构清晰，5 分
- 没有明显大量重复代码，5 分

- 能清楚说明实现思路，5 分
